# Week02 Concept Answer Key

> 請先完成 `concept_practice.md` 的作答，再查看本參考答案。

以下是理解方向，不要求逐字相同。

## 1. CLIP 的基本任務

CLIP 將影像與文字分別編碼成向量，再讓相符圖文在共同嵌入空間中具有較高相似度。常見誤解是把它當成查資料庫或聊天模型；資料庫檢索只是可使用 CLIP 向量的一種系統應用。

## 2. Image Encoder 與 Text Encoder

Image Encoder 接收影像並輸出影像表示；Text Encoder 接收 Token 化文字並輸出文字表示。兩者通常再投影成相同維度，才能計算跨模態相似度。

## 3. 共同向量空間

共同向量空間是讓不同模態輸出可用同一種距離或相似度比較的表示空間。重點不是「儲存在哪裡」，而是模型學到哪些方向代表相近語意。

## 4. Cosine Similarity

餘弦相似度比較兩個向量的方向是否接近，並用向量長度做正規化。高分表示在模型表示中較匹配，不代表客觀事實或完整理解。

## 5. Zero-shot Classification

先把候選類別寫成 prompt，將圖片與每個 prompt 編碼成向量，再計算相似度。分數可經 Softmax 形成候選集合內的相對機率，最高者成為 top-1 預測。提示詞與候選集合改變時，結果也可能改變。

## 6. Contrastive Learning

訓練時提高正確圖文配對的相似度，降低錯誤配對的相似度。模型因此同時調整影像與文字表示，而不是只記住單一類別答案。

## 7. CLIP 與一般分類模型

一般分類模型常有固定輸出類別與分類頭；CLIP 可把新候選類別寫成文字並比較相似度，因此能做零樣本分類。但「zero-shot」不表示模型從未接觸相關概念，其能力仍來自大規模預訓練資料。

## 8. CLIP 與 VLM / VLA

CLIP 類圖文對齊能力可支援視覺語意表示、檢索與候選排序。VLM 進一步加入語言生成與推理，VLA 再把視覺、文字和機器人狀態連到動作決策。CLIP 是前置能力之一，不等於完整系統。
